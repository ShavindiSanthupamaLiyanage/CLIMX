User Profile

Technical Description

The Profile Page is a responsive and accessible user profile layout built using HTML and CSS. It follows a well-structured three-column layout, ensuring clear organization and readability. The design adapts to different screen sizes for seamless usability on various devices.

  • Left Section – Contains a profile card with the user’s image, personal details, and a skills section.
  • Middle Section – Includes a biography, showcasing a short introduction about the user, along with achievements.
  • Right Section – Displays volunteer activities, UN goals.
  • Experience Section – Placed below the main layout, listing work experience in a structured format.
  • Contact Me Card – Also positioned below the main layout, providing an interactive contact form.

    • Accessibility

    This profile page is designed with accessibility best practices.

  • Semantic HTML elements to improve screen reader compatibility.
  • High contrast color combinations for readability, with a green and white palette that ensures good visibility.
  • Hover and focus effects enhance usability for users
  • Responsive design adapts to different screen sizes, ensuring accessibility on mobile devices.

    • Sitemap

    Technical Description

    The Sitemap Page of the CLIMX project is a structured and interactive visual representation of the website's navigation. It is built using HTML, CSS, and SVG to ensure clarity and accessibility.

  • The sitemap is rendered using SVG to create a graphical structure of the website’s pages.
  • Interactive Nodes: Each page is represented by a rec. SVG node, which changes color when hovered over.
  • Connecting Lines: Black lines link related pages, visually representing the website hierarchy.
  • The sitemap is placed within a visually appealing card with a gradient background, rounded corners.
  • CSS Flexbox: Used to center the sitemap.
  • Hover Effects: Interactive SVG nodes change color for better user engagement.
  • Typography: The text inside SVG nodes is bold and white for readability.

    • Accessibility

    The Sitemap Page follows best practices in web accessibility to ensure inclusivity for all users, including those with disabilities.

    1. Semantic HTML
      • Proper use of tags to ensures a well-structured and readable layout for screen readers.
    2. Keyboard Navigation
      • The navigation menu and sitemap nodes are fully keyboard-accessible.
      • Users can navigate using the Tab key and activate links with the Enter key.
    3. Color Contrast & readability
      • The background and text colors are carefully selected for high contrast to improve readability.
      • text inside SVG elements ensures clarity against the dark green background.
    4. Hover & Focus states
      • CSS hover effects provide visual feedback for mouse users.
      • Keyboard focus states allow screen reader users to navigate effectively.

    Content Page

    Technical Description

    The page is designed to offer an intuitive and visually appealing user experience while ensuring its functionality across various devices. The content is divided into distinct sections that focus on key topics. The layout uses modern design techniques to create a flexible and responsive structure.

    1. Layout and Structure
      • The design employs a flexible layout system that ensures elements such as images and text are displayed in a way that adapts to different screen sizes.
    2. Navigation
      • A sticky navigation menu at the top of the page allows users to quickly access different sections of the page without needing to scroll back to the top
    3. Visual Styling
      • A color palette is carefully chosen to reflect the website's environmental theme, with green and earth tones dominating the design. This reinforces the page's message while ensuring that text stands out clearly against the background.
    4. Buttons and links
      • Interactive elements like buttons and links are styled to be prominent and easy to click or tap. They are designed to stand out visually while maintaining a cohesive look with the rest of the site. The "Go to Top" button is fixed in place, offering users quick access to the top of the page with a single click, improving overall navigation.

    Accessibility

    The design and structure of the page focus on making it usable for all individuals, including those with disabilities, by adhering to accessibility best practices.

    1. Semantic Structure
      • The layout is organized logically to ensure that users can navigate the page easily
    2. Image Descriptions
      • All images are accompanied by descriptive text that explains their content. This ensures that users with visual impairments, who rely on screen readers, can understand the meaning behind the images.
    3. Color contrast
      • High contrast between text and background colors ensures that the content is readable for users with visual impairments, including those with color blindness or low vision.

    Challenges and Lessons Learned

    1. SVG Interactivity in Sitemap Page
      • Challenge: Implementing interactive hover effects and focus states for SVG elements while maintaining accessibility.
      • Solution: Used CSS styling to improve usability and ensure keyboard navigability.
    2. Integrating a Seamless Navigation Experience
      • Challenge: Implementing a sticky navigation bar without obstructing the user experience.
      • Solution: Adjusted positioning and z-index properties to keep the navigation accessible without covering content.
    3. Testing and Validation Issues
      • Challenge: Errors found during validation and cross-browser testing
      • Solution: Used validation tools (HTML, CSS validators) to fix compatibility issues.
