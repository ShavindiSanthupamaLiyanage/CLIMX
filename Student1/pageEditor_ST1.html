Splash Page

The splash screen serves as the introductory page for the website, featuring a brief animation and an automatic redirect to the content page after 4 seconds.

Technical Description

HTML Techniques Used

  • Automatic Redirection: The page uses <meta http-equiv="refresh" content="4;url=../Content/content1.html"> to redirect users after 4 seconds.
  • Layout Structure: The main container <div class="splash-container"> ensures structured content alignment.
  • Branding Elements: Displays the CLIMX logo using <img src="images/logo.png"> and a large title <h1 class="title">.
  • Mission Statement: Text is presented in <p class="mission"> for clarity and separation.
  • Team Members: Structured list created with <ul> and <li> elements.
  • Loading Animation: The <div class="loader"> contains an animated loading effect using CSS keyframes.

CSS Techniques Used

  • Background Styling: The splash screen applies a full-screen background using background: url('images/splash.png') no-repeat center center; with background-size: cover; for proportional scaling.
  • Overlay Effect: A semi-transparent overlay is added using body::before { background: rgba(0, 0, 0, 0.5); } to enhance text readability.
  • Loader Animation: The blinking dots animation is achieved through @keyframes blink and applied with animation: blink 1.5s infinite;.

Accessibility

The splash screen follows accessibility best practices:

  • Text Readability: The overlay effect ensures sufficient contrast between text and background.
  • Alternative Text: The CLIMX logo includes alt="CLIMX Logo" for screen readers.
  • Keyboard Navigation: No unnecessary auto-focus elements, ensuring smooth navigation.
  • Time Consideration: A 4-second delay provides enough time for users to recognize the content before redirection.

Volunteer Page

The volunteer page showcases various climate action programs and allows users to sign up for opportunities.

Technical Description

HTML Techniques Used

  • Header: Displays the page title "Climate Action Volunteering" using <h1>.
  • Banner Section: Uses <div class="banner-overlay"> for an overlay effect, with introductory text.
  • Cards Section: Displays three program cards with an icon, title, description, and sign-up button.
  • Table Section: Lists upcoming programs using a <table> for structured display.
  • Volunteer Programs Section: Contains a dropdown filter (<select>) and program cards.
  • Stats Banner: Uses .stats-item components to highlight volunteering impact.
  • Review & Rating: Features a review panel with dynamic review cards.
  • Feedback Form: Allows users to submit reviews with a <textarea> and star ratings.
  • Sign-Up Modal: Provides an interactive form for volunteer registrations.

CSS Techniques

  • Layout & Positioning: Uses display: flex; and justify-content: center; for alignment.
  • Typography & Styling: Applies text-transform: uppercase; for banners and box-shadow for modern card design.
  • Responsive Design: Uses media queries and flex-wrap: wrap; for mobile adaptability.

JavaScript Functionality

  • Program Filtering: Uses an event listener to filter programs dynamically.
  • Modal Handling: Controls opening and closing of the sign-up modal.
  • Review Toggle: Expands and collapses review sections dynamically.

Accessibility

Ensuring accessibility on the volunteer page:

  • Alt Text: All images include descriptive alt attributes.
  • Keyboard Navigation: Interactive elements are focusable.
  • Form Labels: Input fields include <label> elements for clarity.

Content Page

The Content Page provides an overview of international climate agreements, policies, and events, including key treaties like the Kyoto Protocol, Paris Agreement, and Montreal Protocol. It highlights major UN Climate Conferences (COPs) and global climate events, offering structured information with interactive details and smooth navigation.

Technical Description

HTML Techniques Used

  • Semantic HTML Elements: Used <section>, <nav>, <header>, <article> for better document structure and accessibility.
  • Fixed Navigation Bar: Implemented internal links using <a href="#id"> for smooth section navigation.
  • Structured Headings: Used <h1>, <h2>, <h3> to maintain readability and SEO.
  • Embedded Images: Used <img> with alt attributes for accessibility and proper scaling within sections.
  • Flexible Content Containers: Applied <div> elements to group text, images, and interactive components.
  • Read-More Functionality: Added hidden <div class="details-box"> sections that expand on click.
  • Navigation Menu: Used <ul> and <li> inside <nav> for an accessible and structured navigation menu.
  • Responsive Layout: Ensured responsive layout using nested <div> elements and class-based styling.
  • Text Emphasis: Applied <strong> and <span> to highlight key information.
  • Content Flow: Maintained content flow and readability by structuring text within <p> tags instead of using excessive line breaks.

CSS Techniques Used

  • Fixed Navigation: Implemented a fixed navigation bar with position: fixed; top: 0; and a z-index for visibility.
  • Flexbox Layout: Used display: flex; and justify-content: space-between; for responsive design.
  • Box-Shadow & Border-Radius: Applied to improve the visual aesthetics of cards.
  • Button Hover Effects: Styled buttons with hover effects using transform: scale(1.05); for smooth transitions.

JavaScript Techniques Used

  • Navigation Scroll Highlighting: Used scrollY to detect the active section and apply a highlight class to the corresponding navigation link.
  • Smooth Scrolling: Implemented smooth scrolling to sections with window.scrollTo({ top: y, behavior: "smooth" });.
  • Dynamic Read More Details: Implemented openDetails(event, id) and closeDetails(event, id) to toggle detailed information.

Accessibility

Best practises to ensure accessibility on the content page:

  • Alt Attributes for Images: Used alt attributes for images to improve screen reader compatibility.
  • High Contrast Colors: Ensured high contrast colors for readability, making sure text stands out against the background.
  • Keyboard-Friendly Navigation: Implemented keyboard-friendly navigation to ensure all interactive elements are accessible via keyboard.

Challenges and Lessons Learned

Challenges & Solutions

  • Challenge: Ensuring the splash screen loads correctly across different screen sizes.
  • Solution: Used background-size: cover; to make the splash screen responsive.
  • Challenge: Implementing a smooth transition with the auto-redirect.
  • Solution: The <meta http-equiv="refresh"> tag enabled an automatic redirect without requiring JavaScript.
  • Challenge: Ensuring proper alignment of elements across screen sizes.
  • Solution: Used display: flex; and media queries for layout responsiveness.
  • Challenge: Handling modal interactions efficiently on the volunteer page.
  • Solution: Used event listeners to manage modal state (open/close) dynamically.
  • Challenge: Ensuring navigation remained fixed without overlapping content.
  • Solution: Added padding-top: 100px; to prevent overlap of fixed navigation bar.
  • Challenge: Smooth scrolling with an offset on the content page.
  • Solution: Used yOffset = -80; to align the scrolling with the navigation height.
